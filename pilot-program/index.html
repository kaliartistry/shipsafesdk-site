Pilot Funnel

ShipSafe SDK Pilot Program

A limited, measurable pilot for cruise lines that want to test Return-to-Ship Safety in an existing passenger app before broader rollout.

Discuss a Pilot → View SDK tiers
Pilot Shape

Validate integration, alerts, and operating value.

The pilot is designed to prove feasibility and gather partner-specific evidence, not to assume fleet-wide ROI before the data exists.

Ideal partner

A cruise line, cruise technology platform, or shore excursion platform with an existing passenger app and clear access to port-call timing data.

Core pilot scope

Offline walking ETA, return-to-ship alert thresholds, selected port data, GoTime meeting-point logic where excursion data is available, and pilot reporting.

Success criteria

Integration effort, offline behavior, alert timing, passenger-facing copy, operations feedback, and close-call signal quality.

01 · ALIGN

Confirm pilot assumptions

Choose the ports, app surface, alert language, operational buffers, and pilot reporting expectations.

02 · INTEGRATE

Embed the SDK

Connect the host app to the SDK with the minimum data needed for walking ETA and alert behavior.

03 · TEST

Run controlled scenarios

Validate offline routes, alert thresholds, meeting-point handling, and fallback behavior before passenger exposure.

04 · DECIDE

Review pilot evidence

Use the pilot report to decide whether to expand ports, extend GoTime, or prepare a broader deployment plan.
